What is Sahil-Code?

An AI coding agent that lives in your terminal. It reads, writes, and edits code using any LLM you choose. Here's what each part does.

The Conversation Loop

At its core, sahil-code is an agentic loop. You give it a task in natural language, and it plans, executes tools, and iterates until the job is done. The agent decides which tools to call, interprets results, and handles errors autonomously.

  • Multi-turn conversation with full context
  • Automatic tool selection and chaining
  • Up to 16 iterations per turn (configurable)
  • Markdown-rendered responses with syntax highlighting

The Tool System

10 built-in tools give the agent hands to work with your codebase. Each tool produces structured JSON output with metadata like duration, byte counts, and line numbers.

  • bash — Run shell commands with timeouts and background mode
  • read_file / write_file / edit_file — Precise file operations
  • glob_search / grep_search — Find files and content
  • list_directory / diff_files / web_fetch — Explore and compare
▶

Any Model, One Interface

Swap between Anthropic Claude, OpenAI GPT, local Ollama models, or any OpenAI-compatible API with a single flag. The same tools, the same conversation format, the same workflow.

  • --provider anthropic with automatic key detection
  • --provider ollama for fully local, private coding
  • Fallback chains: try Claude first, fall back to GPT on failure
  • Per-model cost tracking and capability detection
▶
Demo: Provider Switching

Safety & Permissions

You decide what the agent can do. The permission system ranges from fully autonomous to interactive approval for every action. A filesystem sandbox prevents path traversal, and dangerous commands are blocked by default.

  • Three modes: allow-all, read-only, prompt
  • Filesystem sandbox restricts operations to your project
  • Bash command blocklist for dangerous patterns
  • Full audit trail of every tool execution
▶
Demo: Permission Prompts

Sessions & History

Every conversation is automatically saved. Resume where you left off, browse history, export as markdown or HTML, and share your sessions with teammates.

  • Auto-save after every turn to ~/.claw/sessions/
  • --resume last picks up your most recent session
  • Automatic compaction keeps context windows efficient
  • Export as styled HTML or markdown for documentation
▶
Demo: Session Resume

Plugins & Extensions

Add new tools, connect to MCP servers, and hook into the execution pipeline. The plugin system makes sahil-code infinitely extensible without modifying core code.

  • Drop-in plugin directories with plugin.json manifests
  • MCP client for connecting to external tool servers
  • Pre/post hooks for custom logic around tool execution
  • Bundled skills: verify, commit, refactor, explain
▶
Demo: Plugin Loading

Multi-Agent Orchestration

For complex tasks, spawn sub-agents that work in parallel on different parts of your codebase. The orchestrator manages coordination, file locks, and result aggregation.

  • Sub-agent spawning with independent token budgets
  • Parallel task execution across files
  • File lock management to prevent conflicts
  • Plan-execute pattern for structured multi-step work
▶
Demo: Multi-Agent Tasks

Rich Terminal Experience

More than a plain text interface. Syntax-highlighted markdown, bordered tool panels, contextual progress spinners, input history, and even vim keybindings for power users.

  • Markdown rendering with pulldown-cmark + syntect
  • Bordered panels for tool input/output
  • Vim mode with normal/insert/visual and motions
  • Line editor with history recall and cursor movement
▶
Demo: Terminal UX

Headless & CI Mode

Run sahil-code in CI/CD pipelines with no interactive input. It outputs structured JSON, auto-approves tools, and exits with appropriate status codes. Pair it with the HTTP API server for remote access.

  • --headless "fix linter errors" for CI pipelines
  • Structured JSON output for machine consumption
  • claw serve for HTTP API with SSE streaming
  • WebSocket support with real-time tool approval
